# 菜單拍照計算機 — 操作說明報告

## 一、三功能首頁

### 📸 拍照辨識菜單
| 步驟 | 說明 |
|------|------|
| 1. 點擊「拍照辨識菜單」 | 前往 Gemini 頁面 |
| 2. 上傳菜單照片給 Gemini | Gemini 回傳 JSON 格式的辨識結果 |
| 3. 複製 Gemini 回傳的 JSON | 回到本工具 |
| 4. 貼上 JSON → 按「解析並編輯品項」 | 品項自動填入編輯頁 |
| 5. 檢查 / 修改品項 | 可增減修改 |
| 6. 按「開始點餐」 | 進入計算機 |

### 🧮 直接開始點餐
| 步驟 | 說明 |
|------|------|
| 1. 點擊「直接開始點餐」 | 進入計算機（使用預設品項或上次儲存的內容） |
| 2. 增減數量、查看總金額 | 離線操作，存於 localStorage |

### 🏪 新增店家
| 步驟 | 說明 |
|------|------|
| 1. 點擊「新增店家」 | 開啟新增表單 |
| 2. 輸入序號（4-6 碼英文+數字）、店家名稱、密碼 |  |
| 3. 設定使用期限（天數或指定到期日） |  |
| 4. 按「新增」 | 寫入 Firestore `mpc-store-codes`，顯示於下方店家清單 |
| 5. 店家清單 | 可修改期限（點 ✏️）、刪除序號 |

## 四、序號與店家管理（Firestore `mpc-store-codes`）

### 管理員操作
| 動作 | 方式 |
|------|------|
| 新增序號 | 首頁 → 新增店家 |
| 設定期限 | 新增時選天數或指定到期日 |
| 修改期限 | 店家清單 → 點 ✏️ → 輸入新天數或日期 |
| 刪除序號 | 店家清單 → 點 ✕ |

### 資料結構
```
mpc-store-codes/{customCode}
  ├── storeName: string    店家名稱
  ├── password: string     密碼（不限長度）
  ├── linkDurationDays: number | null  期限天數（0=永久）
  ├── linkExpireDate: Timestamp | null 指定到期日
  ├── forcePasswordChange: boolean     首次登入強制改密碼
  └── createdAt: Timestamp
```

## 五、短網址系統（Firestore `mpc-shortlinks`）

### 運作原理

| 情境 | 行為 |
|------|------|
| 首次產生網址 | 建立新文件 docId = `序號-亂數8碼` |
| 再次更新內容 | 找到同一序號的既有文件 → 覆蓋內容 → **網址不變** |
| 已散佈的舊網址 | ✅ 永遠可用（內容永遠最新） |

### 流程
```
管理員 / 店家
    │
    ├─ 修改品項 / 主題 / 文字
    │
    ├─ 按「產生網址」或「更新內容」
    │     ├─ 有既有文件 → 更新 Firestore（同 docId，同 URL）
    │     └─ 無既有文件 → 建立新 Firestore 文件（新 docId，新 URL）
    │
    └─ 取得網址 → 傳給客人
         │
         └─ 客人開網址 → 載入 Firestore → 顯示最新內容

網路需求：
  管理員產生網址：✅ 需要
  店家更新內容：✅ 需要
  客人首次打開：✅ 需要
  客人點餐：❌ 離線即可
```

### 期限檢查
- 客人開啟網址時檢查 `expiresAt`
- 過期 → 顯示「此連結已過期」
- 離線期間不過期（不連網就不檢查）
- 期限延長只在店家清單（✏️ 修改)

### 資料結構
```
mpc-shortlinks/{docId}
  ├── customCode: string    店家序號
  ├── randomCode: string    亂數 8 碼
  ├── title / subtitle / footer / ...  頁面設定
  ├── colors: { bodyBg, headerBg, ... } 配色
  ├── items: [{ id, name, price, unit, category }]  品項
  ├── createdAt: Timestamp  首次建立時間
  ├── updatedAt: Timestamp  最後更新時間
  └── expiresAt: Timestamp  到期時間
```

## 六、權限與商店模式

| 角色 | 判別方式 | 可操作範圍 |
|------|----------|-----------|
| 管理員（你） | `?s=` 不存在（直接開頁面） | 全部功能 ＋ 三功能首頁 |
| 店家 | `?s=` 存在 ＋ 輸入密碼解鎖 | 設定頁（無 JSON/文字匯入）＋ 更新內容 ＋ 複製連結 |
| 客人 | `?s=` 存在，未解鎖 | 只能點餐，無法進入設定 |

## 七、Firestore 集合一覽

| 集合名稱 | 用途 | 誰寫入 | 誰讀取 |
|----------|------|--------|--------|
| `mpc-store-codes` | 店家序號、密碼、期限設定 | 管理員（新增店家） | 管理員 ＋ 店家（密碼驗證） |
| `mpc-shortlinks` | 短網址內容 | 管理員 ＋ 店家（產生／更新） | 所有人（開啟網址） |

### 用量評估（100 家店，每日更新一次）
| 項目 | 每次 | 每日總計 |
|------|------|---------|
| 讀取 store-codes | 1 次 | 100 次 |
| 寫入 store-codes | 0 次（僅管理員操作） | 極少 |
| 讀取 shortlinks（產生前查詢） | 1 次 | 100 次 |
| 寫入 shortlinks | 1 次 | 100 次 |
| 客人讀取 shortlinks | 每家 N 次 | 視客人數量 |
| Firestore 免費額度讀取 | — | 50,000 次/天 |
| Firestore 免費額度寫入 | — | 20,000 次/天 |

## 八、注意事項

1. **內容更新不上傳**：店家或管理員在設定頁修改後，需按「產生網址」或「更新內容」才會寫入 Firestore，否則他人開網址看不到變更
2. **離線可用**：所有資料載入後存於 localStorage，後續操作完全離線，不須連網
3. **期限僅連網時檢查**：不連網重整就不會觸發過期檢查
4. **密碼存雲端**：店家密碼存於 `mpc-store-codes`，非 localStorage
5. **同序號僅一個有效 URL**：多次更新共用同一個 Firestore 文件，舊網址不失效
